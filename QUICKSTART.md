# 快速开始指南 🚀

5分钟快速上手AI漏洞分析助手！

## 第一步：编译插件（2分钟）

### Windows用户（最简单）
```bash
# 1. 打开命令提示符
# 2. 进入项目目录
cd E:\HackTools\develop\PluginExample\untitled

# 3. 运行编译脚本
compile.bat
```

### 其他方式
```bash
# 使用Maven
mvn clean package

# 使用Gradle
gradle clean jar
```

## 第二步：加载插件（1分钟）

1. 打开 **Burp Suite**
2. 进入 **Extensions** → **Installed**
3. 点击 **Add** 按钮
4. 选择 **Java** 类型
5. 选择编译好的JAR文件：
   - 批处理脚本：`build\libs\ai-analyzer.jar`
   - Maven：`target\ai-analyzer-1.0.0-jar-with-dependencies.jar`
   - Gradle：`build\libs\ai-analyzer-1.0.0.jar`
6. 点击 **Next**

✅ 看到"AI漏洞分析助手插件已成功加载！"表示成功

## 第三步：配置API（1分钟）

### 获取通义千问API Key

1. 访问：https://dashscope.console.aliyun.com/
2. 登录/注册阿里云账号
3. 开通DashScope服务（有免费额度）
4. 创建API Key并复制

### 在插件中配置

1. 在Burp Suite中找到 **AI分析** 标签页
2. 填写配置：
   - **API URL**：`https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions`
   - **API Key**：粘贴你的API Key

## 第四步：开始使用（1分钟）

### 方式1：从Proxy发送

1. 在 **Proxy** → **HTTP history** 中找到一个请求
2. **右键** 点击请求
3. 选择 **发送到AI分析**
4. 在AI分析标签页中点击 **开始分析**

### 方式2：从Repeater发送

1. 在 **Repeater** 中打开一个请求
2. **右键** 点击请求区域
3. 选择 **发送到AI分析**
4. 填写提示词，点击 **开始分析**

### 方式3：手动输入

1. 直接在 **AI分析** 标签页的"HTTP请求"区域粘贴请求
2. 在"用户提示词"区域输入分析需求
3. 点击 **开始分析**

## 示例场景

### 场景1：快速安全检查

**HTTP请求**：
```
GET /api/user?id=123 HTTP/1.1
Host: example.com
Cookie: session=abc123
```

**用户提示词**：
```
快速分析这个请求的安全风险，重点关注SQL注入和越权访问
```

**预期结果**：
- AI会分析id参数的SQL注入可能性
- 检查session的安全性
- 评估越权访问风险
- 给出测试建议

### 场景2：深度漏洞挖掘

**用户提示词**：
```
请详细分析这个登录请求：
1. 认证机制的安全性
2. 可能的绕过方法
3. SQL注入测试点
4. 推荐5个测试payload
5. 按严重程度排序
```

**预期结果**：
- 详细的安全分析报告
- 具体的测试步骤
- 优先级排序的漏洞列表

### 场景3：特定漏洞检测

**用户提示词**：
```
假设这是一个PHP应用，使用MySQL数据库。
请提供针对性的SQL注入payload，包括：
- 基于时间的盲注
- 基于布尔的盲注
- Union注入
每种至少3个payload
```

**预期结果**：
- 针对MySQL的特定payload
- 详细的注入技巧
- 预期的响应特征

## 提示与技巧

### 💡 提示词技巧

1. **具体明确**：说明要分析的具体漏洞类型
2. **提供上下文**：说明应用类型、技术栈
3. **要求示例**：让AI提供具体的payload
4. **分步骤**：复杂问题拆分为多个小问题

### 💡 效率技巧

1. **保存常用提示词**：将常用的提示词保存到文本文件
2. **批量分析**：对相似请求使用相同的提示词
3. **迭代优化**：根据结果调整提示词，逐步深入
4. **结合工具**：AI分析 + Repeater测试 = 最佳效果

### 💡 安全提示

1. **脱敏处理**：删除请求中的敏感信息（密码、token等）
2. **测试环境**：优先在测试环境使用
3. **人工验证**：AI建议需要人工验证
4. **成本控制**：注意API调用次数

## 故障排除

### ❌ 插件加载失败

**检查**：
- Java版本是否为21+
- JAR文件是否完整
- 查看Burp Suite的Errors标签页

### ❌ API调用失败

**检查**：
- API Key是否正确
- 网络是否可以访问阿里云
- 查看Output标签页的错误信息

### ❌ 分析结果不理想

**优化**：
- 提供更详细的提示词
- 简化HTTP请求内容
- 尝试不同的提问方式
